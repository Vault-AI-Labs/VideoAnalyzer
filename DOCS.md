# VideoTranscriber

Drop videos, extract transcripts and frames for AI.

## What It Does

Local web app that takes any video file (TikTok downloads, screen recordings, tutorials) and produces:
- Full transcript via OpenAI Whisper (runs locally)
- Key frames extracted at smart intervals via ffmpeg
- A "Copy for Claude" button that packages everything into a paste-ready prompt with `read` commands for every frame

## Architecture

```
~/Desktop/VideoTranscriber.app    # macOS .app launcher (double-click)
~/PROJECTS/VideoTranscriber/      # Source code
~/.video-transcriber/             # Runtime data
  ├── port                        # Current port (written on startup)
  ├── videotranscriber.log        # Server log
  ├── uploads/                    # Temporary (deleted after processing)
  └── outputs/<job_id>/           # Persisted results
      ├── metadata.json           # All job data + claude_prompt
      ├── transcript.txt          # Clean transcript
      └── frames/frame_*.jpg      # Extracted key frames
```

### Stack

| Component | Technology |
|-----------|-----------|
| Backend | Python Flask (single file `app.py`) |
| Frontend | Single-page HTML (`templates/index.html`), vanilla JS, no build step |
| Transcription | OpenAI Whisper (brew install, runs locally) |
| Frame extraction | ffmpeg |
| Title generation | Ollama `gemma3:4b` (local LLM, heuristic fallback if unavailable) |
| Launcher | macOS .app bundle with bash launcher script |
| Dependencies | `uv run --with flask` (zero venv setup) |

### Port Sniffing

The app tries port 7745 first, then walks 7746-7799 if occupied. The chosen port is written to `~/.video-transcriber/port`. The launcher reads this file to know where to open the browser. Double-clicking again detects the running server and just opens a new tab.

## Key Features

### Thumbnails
Each transcription in the history list shows a thumbnail from the first extracted frame, giving a quick visual reference for each video.

### Status Tracking
Each transcription has a status: **New**, **Doing It**, **Skipped**, **Revisit Later**, **Delete**. Changeable from both the detail view and the home screen history list (dropdown pinned to the right). Dates are tracked.

### LLM-Generated Titles
Transcription titles are generated by `gemma3:4b` via Ollama to produce article-style headlines (e.g., "OpenClaw Felix Bot: $80k in 30 days"). Falls back to heuristic word extraction if Ollama isn't running.

### Copy for Claude
The "Copy for Claude" button produces a pre-built prompt (generated server-side with verified absolute paths) that:
1. Includes the full transcript inline
2. Lists `read` commands for every extracted frame
3. Instructs Claude to explain the video first, then ask what to do

### Whisper Model Selection
Dropdown in the header: tiny (fastest) through large (best accuracy). Default is `base`.

## API Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/` | Serve the web UI |
| POST | `/api/upload` | Upload video, returns `job_id` |
| GET | `/api/stream/<job_id>` | SSE stream of processing progress |
| GET | `/api/frames/<job_id>/<file>` | Serve extracted frame image |
| GET | `/api/history` | List all past transcriptions |
| PUT | `/api/status/<job_id>` | Update item status (doing/skipped/revisit/delete) |

## Processing Pipeline

1. **Upload** — video saved to `~/.video-transcriber/uploads/<job_id>.<ext>`
2. **Analyze** — ffprobe extracts duration
3. **Transcribe** — Whisper processes audio (model selectable). Heartbeat events sent every 30s to prevent SSE timeout on long videos.
4. **Extract frames** — ffmpeg pulls key frames at smart intervals (heartbeat-protected):
   - <60s video: every 3 seconds
   - 60-180s: every 10 seconds
   - >180s: every 20 seconds
   - Capped at 24 frames
5. **Generate title** — Ollama `gemma3:4b` produces 5-7 word headline
6. **Build Claude prompt** — Server-side, with verified absolute frame paths
7. **Save metadata** — All data persisted to `metadata.json`
8. **Cleanup** — Upload video deleted, whisper temp files removed

## Running

### Desktop (primary)
Double-click `VideoTranscriber.app` on Desktop.

### CLI
```bash
~/PROJECTS/VideoTranscriber/start.sh
```

### CLI tool (standalone)
```bash
vdump video.mp4 [model]
# Output: video_dump/transcript.md + frames/
```

## Dependencies

Installed via Homebrew:
- `openai-whisper` — local transcription
- `ffmpeg` — video/frame processing
- `ollama` + `gemma3:4b` model — title generation (optional)

Installed via `~/.local/bin/uv`:
- `flask` — web framework (auto-installed on each run via `uv run --with flask`)
